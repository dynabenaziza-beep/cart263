Project II Response

Project 1

Link: https://xp30n.github.io/SAVE-THE-INTERNET/
This project really caught my attention because it shows different versions of the internet. At the start, it looks like an old computer desktop with pixel visuals and classic icons. It made me think about the early internet. It felt simple and nostalgic, like when the internet was more personal.
Then it changes into a 3D space with images floating around. This part was interesting because it felt more immersive, like I was inside the internet. One thing that stood out to me was the webcam in the corner. It made me realize that I was part of the system, not just watching it.
There is also a moment where you click on an icon and a page opens showing your own face with the webcam. This felt very uncomfortable. I felt like I was being watched. It reminded me of what we talked about in another class about privacy on the internet. Usually, we don’t really notice when we are being tracked, but here it is very clear. That makes the idea stronger.
I think the project is about how the internet changed over time. The title “Save the Internet” makes it feel like something is lost. The difference between the old desktop and the 3D space shows how the internet became more complex and maybe more controlled.
Project 2

Link: https://lee-sifu.github.io/Project-2_Cart263/Project-2_Cart263-main/
At first, this project feels very simple. You click on a cube and the number goes up. The background is dark with small moving particles, which looks clean and a bit futuristic.
But after clicking more, things start to change. The background changes color, random images appear, and everything becomes more chaotic. The moment when the message “YOU ARE BREAKING THE SYSTEM!” appears was very strong. It made me feel like my actions had an impact.
I think this project is about how too much interaction can break a system. A simple action like clicking becomes too much, and the system reacts in a chaotic way. It shows how people use technology a lot without thinking about  the limits.
One thing I found really interesting in the code is that one click does many things at the same time. It changes colors, creates particles, and sometimes shows images. Every 5 clicks, a random image appears, which makes it less predictable. After around 30 clicks, the cube breaks into smaller pieces, which is surprising. This shows that the code is using loops and many objects to create that effect. There is also a reset after 50 clicks, which feels like the system is overloaded and starts again.

Project 3

Link: https://philippe-bn.github.io/cart263/projects/project-2/
This project is more about visuals and atmosphere. When I opened it, I saw a landscape with mountains and a sky full of flying shapes that look like birds. There are many of them moving at the same time.
This project feels more like watching than interacting. You don’t really control anything, you just observe what is happening. It feels calm at first, but also a bit overwhelming because there are so many moving elements.
What I found interesting is the repetition. The birds look peaceful at first, but because there are so many, it can also feel a bit chaotic . That mix makes the project more interesting.


Overall Reflection
These three projects are all very different. One is about the internet and privacy, one is about interaction and breaking a system, and one is about visuals and atmosphere.
What I learned is that small choices in code can really change how a project feels. Things like repetition, interaction, and timing are very important. It made me think about my own project and how I can make my idea clearer.
Overall, these projects show that code is not just technical, it also helps create meaning and experience
